一、项目概述
================================================================================

本项目是一个基于 RAG（检索增强生成）架构的"智能服装客服系统"。
核心功能：用户提问后，系统先从本地知识库检索相关文档片段，再结合大模型生成
专业回答。知识库覆盖三个方面：

  ● 尺码推荐 — 根据身高体重推荐服装尺码（S~5XL）
  ● 洗涤养护 — 按面料材质、季节提供洗护指导
  ● 颜色选择 — 按肤色、场合、体型、季节推荐颜色搭配

系统支持：
  ● 多轮对话（聊天历史自动保存/加载）
  ● 流式输出（回答逐字显示）
  ● 知识库动态更新（上传 .txt 文件自动入库）

四、架构与数据流
================================================================================

4.1 知识入库流程（app_file_uploader.py + knowledge_besa.py）

  用户上传 .txt 文件
    │
    ▼
  [1] MD5 去重检查 ──→ 已存在 → 跳过，返回"内容已经存在"
    │ 未入库
    ▼
  [2] RecursiveCharacterTextSplitter 文本切片
      ● chunk_size = 1000 字符
      ● chunk_overlap = 100 字符
      ● 按优先级切分：段落 → 换行 → 句号 → 感叹号 → 问号 → 空格 → 逐字
    │
    ▼
  [3] DashScope text-embedding-v4 生成向量
    │
    ▼
  [4] 向量 + 元数据存入 ChromaDB
      ● 元数据：source（文件名）、create_time（入库时间）、operator（操作人）
    │
    ▼
  [5] 记录 MD5 指纹到 .md5.text

4.2 问答流程（app_qa.py + rag.py + vector_stores.py）

  用户输入问题
    │
    ▼
  [1] 向量检索（ChromaDB similarity search）
      ● 把用户问题用 DashScope 转成向量
      ● 在 ChromaDB 中找最相似的 top-2 个文档片段
    │
    ▼
  [2] 组装 Prompt
      ● System: 角色设定 + 检索到的参考文档片段
      ● History: 当前会话的聊天历史（多轮对话上下文）
      ● User: 用户当前提问
    │
    ▼
  [3] DeepSeek LLM 流式生成回答
    │
    ▼
  [4] 保存聊天历史 → 展示回答（逐字流式输出）

五、核心代码详解
================================================================================

5.1 config_data.py — 全局配置

  集中管理所有可调参数，修改一处全局生效：

  ┌─────────────────────────┬───────────────────────────┬────────────────┐
  │ 参数                     │ 值                        │ 说明            │
  ├─────────────────────────┼───────────────────────────┼────────────────┤
  │ chunk_size              │ 1000                      │ 切片最大字符数   │
  │ chunk_overlap           │ 100                       │ 切片间重叠字符   │
  │ collection_name         │ "rag"                     │ ChromaDB 集合名  │
  │ persist_directory       │ "./chroma_db"             │ 向量库存储路径   │
  │ similarity_threshold    │ 2                         │ 检索返回 top-k   │
  │ embedding_model_name    │ "text-embedding-v4"       │ 阿里云 embedding │
  │ deepseek_model_name     │ "deepseek-chat"           │ DeepSeek 模型    │
  │ deepseek_base_url       │ "https://api.deepseek.com/v1" │ API 地址    │
  │ session_config          │ session_id: "user_001"    │ 默认会话 ID      │
  └─────────────────────────┴───────────────────────────┴────────────────┘

5.2 rag.py — RAG 核心链（LCEL 管道）

  使用 LangChain Expression Language (LCEL) 管道模式：

  RunnableLambda(取出问题)
    → retriever(向量检索)
    → format_document(格式化检索结果)
    ｜
    ├─ 同时透传 input（RunnablePassthrough）
    ｜
    → format_for_prompt_template(重组数据)
    → ChatPromptTemplate(填入提示词模板)
    → ChatOpenAI(DeepSeek) 流式生成
    → StrOutputParser() 解析为文本

  外层包裹 RunnableWithMessageHistory，自动管理多轮对话历史。

5.3 vector_stores.py — 向量检索

  封装 ChromaDB，对外提供一个 retriever：
  - 调用 .as_retriever(search_kwargs={"k": 2}) 返回 top-2 相似文档
  - 内部使用 Chroma 连接已有数据库，不负责写入

5.4 knowledge_besa.py — 知识库入库

  核心类 KnowledgeBesaService：
  - upload_by_str(data, filename)：将文本入库
    ① MD5 去重（与 .md5.text 比对）
    ② RecursiveCharacterTextSplitter 切片（>1000 字符才切）
    ③ 添加元数据（source, create_time, operator）
    ④ Chroma.add_texts() 批量写入向量库
    ⑤ 记录 MD5 指纹

5.5 file_history_store.py — 聊天历史

  继承 LangChain 的 BaseChatMessageHistory：
  - 每个 session_id 对应一个 JSON 文件（存储在 chat_history/ 目录）
  - add_messages()：追加新消息并序列化
  - messages 属性：读取并反序列化历史消息
  - clear()：清空当前会话

5.6 app_qa.py — 问答 Web 页面

  基于 Streamlit 的聊天界面：
  - st.chat_input() 底部输入框
  - st.chat_message() 渲染气泡
  - chain.stream() 流式生成，逐字显示
  - st.session_state 持久化 RAG 服务和消息列表

5.7 app_file_uploader.py — 知识库上传页面

  基于 Streamlit 的文件上传界面：
  - st.file_uploader() 文件选择组件（仅允许 .txt）
  - 读取文件内容 → 调用 KnowledgeBesaService.upload_by_str()


================================================================================
六、使用教程
================================================================================

6.1 环境准备

  1) 安装 Python 3.10+

  2) 安装依赖库（在项目目录下执行）：
     pip install streamlit langchain langchain-core langchain-community \
                 langchain-openai langchain-chroma chromadb

  3) 配置 API Key（编辑 config_data.py）：

     阿里云百炼 DashScope API Key：
     dashscope_api_key = "sk-你的key"

     DeepSeek API Key：
     deepseek_api_key = "sk-你的key"

     也可以使用环境变量（推荐）：
     export DASHSCOPE_API_KEY="sk-你的key"
     export DEEPSEEK_API_KEY="sk-你的key"

6.2 知识库入库（上传知识文档）

  1) 打开终端，进入项目目录：
     cd D:\项目1RAG\111\项目案例RAG

  2) 启动文件上传页面：
     streamlit run app_file_uploader.py

  3) 浏览器会自动打开 http://localhost:8501

  4) 点击 "Browse files" 按钮，选择 data/ 目录下的 .txt 文件上传：
     - 尺码推荐.txt
     - 洗涤养护.txt
     - 颜色选择.txt

  5) 页面显示 "内容成功载入向量库" 即表示入库成功。
     如果重复上传相同文件，会显示 "跳过，内容已经存在"。

  ⚠️ 注意：如果 API Key 未正确配置，上传时会报错。

6.3 启动智能客服问答

  1) 在终端执行：
     streamlit run app_qa.py

  2) 浏览器打开 http://localhost:8501

  3) 在底部输入框输入问题，例如：
     - "我身高172cm，体重120斤，应该穿什么尺码？"
     - "真丝连衣裙怎么洗？"
     - "黄皮肤适合穿什么颜色的衣服？"
     - "冬天羽绒服怎么保养？"

  4) 系统会：
     ① 从知识库检索最相关的 2 个文档片段
     ② 结合聊天历史上下文
     ③ DeepSeek 生成专业回答（逐字显示）

  5) 支持连续追问，系统会记住之前的对话内容。

6.4 切换会话

  编辑 config_data.py 中的 session_config：
  session_config = {
      "configurable": {
          "session_id": "user_002",  # 改成新 ID 即开启新会话
      }
  }

6.5 修改检索参数

  编辑 config_data.py：
  - similarity_threshold = 3    # 检索更多文档片段（默认 2）
  - chunk_size = 500            # 减小切片尺寸
  - chunk_overlap = 50          # 减小重叠

七、常见问题
================================================================================

Q1: 上传文件时报 API 错误？
A1: 检查 config_data.py 中的 dashscope_api_key 是否正确。推荐用环境变量。

Q2: 问答时没有返回相关知识？
A2: 确保已通过 app_file_uploader.py 上传了知识文档，且 chroma_db/ 目录存在。

Q3: 想清空向量库重新开始？
A3: 删除 chroma_db/ 文件夹和 .md5.text 文件，然后重新上传知识文档。

Q4: 想换一个 LLM？
A4: 修改 config_data.py 中的 deepseek_model_name 和 deepseek_base_url，
    只要兼容 OpenAI 接口的模型都可以（如 OpenAI GPT、本地 vLLM 等）。

Q5: 想换一个 Embedding 模型？
A5: 修改 config_data.py 中的 embedding_model_name，同时修改
    knowledge_besa.py 中的 embedding 初始化代码。

八、技术亮点
================================================================================

1. LCEL 管道模式：代码清晰，链式调用，易于扩展和调试。
2. MD5 去重：避免相同内容重复入库，节省存储和计算成本。
3. 递归切片：按语义边界切分（段落→句子→词→字），保持语义完整。
4. 切片重叠：相邻切片 100 字符重叠，防止关键信息刚好在边界被截断。
5. 流式输出：用户体验好，回答逐字显示，无需等待完整生成。
6. 多轮对话：通过 LangChain RunnableWithMessageHistory 自动管理历史。
7. 配置分离：所有参数集中在 config_data.py，无需到各模块修改。
8. 持久化存储：ChromaDB 存磁盘，程序重启数据不丢失。
